## 使い方

各プロンプトは「タイトル」「ユースケース」「プロンプトテンプレート」「出力例」の構成です。

**Claude Codeで使う際の手順**:
1. プロンプトテンプレートをコピー
2. 括弧内の `[...]` を自分のコード/状況に置き換え
3. Claude Codeの入力欄に貼り付けて実行

**タイトル**: Bug Diagnosis and Fix Assistant

**ユースケース**: エラーが発生しているが原因がわからない時。エラーメッセージとコード片を与えて、原因と修正案を得る。

**プロンプトテンプレート**:
```
以下のエラーが発生しています。原因を診断して、修正方法を提案してください。

[エラーメッセージをコピー]

関連するコード:
[エラーが起きているコード片]

[このコードが何をしようとしているのかの説明]

1. 根本原因 (1-2文で)
2. なぜこのエラーが起きたのか (詳細な説明)

**出力例**:

```javascript

```

テスト方法: 空文字列、"abc"、null、undefined を入力して確認
```

- 深入: 外部への影響、パフォーマンス特性も含める

以下を含めてください:

**出力例**:
```javascript
/**
 * ユーザーリストから特定の条件に合うユーザーをフィルタリングします。
 * 
 * @param {Array<User>} users - フィルタリング対象のユーザー配列
 * @param {Object} filters - フィルタリング条件
 * @param {string} [filters.role] - ロール (例: 'admin', 'user')
 * @param {boolean} [filters.isActive] - アクティブ状態
 * @returns {Array<User>} フィルタリング後のユーザー配列
 * 
 * @example
 * const admins = filterUsers(users, { role: 'admin', isActive: true });
 * 
 * @throws {TypeError} users が配列でない場合
 * 
 * @performance O(n) - 全ユーザーを走査します
 * 
 * @note 大規模データ (100k+ ユーザー) の場合、インデックス活用を検討
 */
function filterUsers(users, filters) {
  // 入力値の検証
  if (!Array.isArray(users)) {
    throw new TypeError('users must be an array');
  }
  
  // filter()を使って条件に合うユーザーを抽出
  // every()で複数条件を AND 判定しています
  return users.filter(user => 
    Object.entries(filters).every(([key, value]) => 
      user[key] === value
    )
  );
}
```

## すぐに使える Tips

### Claude Code との組み合わせ

上記のプロンプトは Claude Code に直接貼り付けて使えます。さらに強力にするために:

1. **ファイル参照**: `/path/to/file.js` でコード内のファイルを自動で読み込める
2. **複数ファイル指定**: 複数ファイルを一気に分析することで、より正確な診断が可能
3. **チェーン実行**: 1つの分析結果を次のプロンプトに渡して、段階的に改善

### 例: リファクタリング流れ

1. 最初に「コード品質スコアリング」で現状把握
2. スコア結果から「トップ3の改善点」を「リファクタリングガイド」で詳細化
3. 「テストケース自動生成」で十分なテスト範囲を確保
4. 修正後のコードを「PR説明文自動生成」で記録
